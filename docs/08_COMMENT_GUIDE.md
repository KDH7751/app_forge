# Comment Guide

## 목적

이 프로젝트에서 주석은 코드를 그대로 설명하기 위한 장치가 아니다.

주석은 다음을 설명한다.

- 구조
- 책임
- 경계
- 의도와 이유

코드만 읽어도 드러나는 내용은 주석으로 반복하지 않는다.

## 기본 원칙

### 무엇이 아니라 왜를 설명한다

잘못된 예:

```dart
// 로그인 상태를 읽는다.
final isLoggedIn = ref.watch(authProvider);
```

올바른 예:

```dart
/// Router는 auth를 직접 읽지 않는다.
/// authState는 app layer에서 주입된다.
```

### 구조와 경계를 먼저 설명한다

중요한 파일은 다음을 분명히 해야 한다.

- 어떤 layer에 속하는지
- 어떤 의존성을 가지는지
- 무엇을 의도적으로 모르는지

### 주석은 설계 문서의 일부다

- 주석은 `docs/01_ARCHITECTURE.md`와 일관되어야 한다.
- 코드 주석은 구조 문서를 보완해야 한다.
- 문서와 주석이 충돌하면 즉시 수정한다.

## 주석 언어 규칙

- 코드 식별자 관련 용어는 영어로 유지한다.
- 설명 문장은 한글로 작성한다.
- 구조와 아키텍처 키워드는 영어로 유지한다.

유지 대상 예시:

- 함수명, 변수명, 클래스명
- state, provider
- Engine, Feature, Router, Plugin, Repository

금지:

- 코드 식별자를 한글로 번역하는 것
- 어색한 직역
- 단어 단위로 깨지는 한글/영어 혼용

## 주석 레벨

### 파일 상단 주석

모든 public 파일은 아래 형식의 파일 상단 주석을 가진다.

```dart
/// ===================================================================
/// [파일 또는 타입 이름]
///
/// 역할:
/// - 이 파일이 하는 일
///
/// 책임:
/// - 이 파일이 담당하는 일
///
/// 경계:
/// - 이 파일이 아는 것과 모르는 것
///
/// 의존성:
/// - 이 파일이 참조할 수 있는 layer
/// ===================================================================
```

### Public API 주석

모든 public 함수와 클래스는 계약을 명시한다.

```dart
/// login을 수행한다.
///
/// 계약:
/// - 성공 시 User를 반환한다.
/// - 실패 시 AppError를 반환한다.
///
/// 규칙:
/// - FirebaseException을 직접 throw하지 않는다.
Future<Result<User>> login(...)
```

### 내부 로직 주석

내부 주석은 최소화한다. 정책이나 예외 처리가 코드만으로 충분히 드러나지
않을 때만 사용한다.

## Layer별 주석 스타일

### Engine

Engine 주석은 설계 중심이어야 한다.

- policy와 extension point를 설명한다.
- Engine이 무엇을 모르는지 적는다.
- 금지된 의존성을 명시한다.

### App

App 주석은 조립 중심이어야 한다.

- 외부 구현이 어떻게 조립되는지 설명한다.
- app이 composition root라는 점을 드러낸다.

### Feature

Feature 주석은 비즈니스 중심이어야 한다.

- slice의 목적을 설명한다.
- flow나 domain 계약을 설명한다.

## 금지 주석

- 코드를 그대로 읽어주는 주석
- 설계 가치가 없는 한 줄 주석
- TODO / FIXME 남발
- 왜가 없는 주석
- 코드와 어긋난 오래된 주석

## AI 협업 규칙

AI가 생성하는 코드는 다음을 반드시 포함해야 한다.

- 파일 상단 주석
- Public API 주석
- 구조와 경계 설명

AI는 다음을 하면 안 된다.

- public 파일에 주석 없이 코드를 생성하는 것
- 구조 설명 없이 구현만 쓰는 것
- Engine 경계 규칙을 주석에 드러내지 않는 것

## 체크리스트

- 파일 상단 주석이 있다.
- Public API 주석이 있다.
- 구조와 경계 설명이 있다.
- 불필요한 주석이 없다.
- 문서와 주석이 일치한다.

## 한 줄 원칙

주석은 코드가 무엇을 하는지 설명하지 않는다.
주석은 왜 이렇게 설계되었는지를 설명한다.
